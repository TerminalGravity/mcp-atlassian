# Tools Reference

MCP Atlassian provides 20 tools for interacting with Jira and Confluence.

## Complete Tool List

### Jira — Read

| Tool | Key Parameters | Description |
|------|---------------|-------------|
| `jira_get` | `keys` (str or list), `response_format` (summary\|full), `include_changelog`, `include_dates`, `include_sla` | Fetch one or many issues. Use `summary` format for lists; `full` for deep inspection. |
| `jira_find` | `jql` or `query` (natural language), `similar_to` (issue key), `limit`, `fields` | Unified search: JQL, natural-language, or similarity-to-a-key. |
| `jira_projects` | `action` (list\|fields\|user), `keyword`, `query` | List all projects, search field schemas by keyword, or look up a user account. |
| `jira_agile` | `action` (boards\|sprints\|sprint_issues\|create_sprint\|update_sprint), `board_id`, `sprint_id` | Boards, sprints, and sprint management in one tool. |
| `jira_handoff` | `keys` (optional list) | Returns a compact, resumable state snapshot — useful before context resets. |
| `jira_worklog` | `key`, `action` (get\|add), `time_spent`, `comment` | Read worklogs on an issue or log time spent. |

### Jira — Knowledge

| Tool | Key Parameters | Description |
|------|---------------|-------------|
| `jira_knowledge` | `query` (natural language), `project`, `limit` | Natural-language Q&A over the synced LanceDB vector index. |
| `jira_vector_sync_status` | — | Reports sync state, last sync time, and indexed issue counts per project. |

### Jira — Write

| Tool | Key Parameters | Description |
|------|---------------|-------------|
| `jira_create` | `project`, `summary`, `issuetype`, `description`, `fields` | Create a new issue. Runs duplicate detection before creating. |
| `jira_update` | `key`, `fields` | Update any field(s) on an existing issue. |
| `jira_assign` | `key`, `assignee` (account ID, email, or `null`) | Assign or unassign an issue. |
| `jira_delete` | `key` | Delete an issue permanently. |
| `jira_transition` | `keys` (one or list), `status` (name), `comment` | Move one or many issues to a status by name. Resolves transition IDs automatically. |
| `jira_comment` | `key`, `body`, `comment_id` (for edit), `body_format` | Add or edit a comment. Response includes a stored-body preview — no follow-up fetch needed. |
| `jira_link` | `action` (add_epic\|remove_epic\|add_web\|add_issue\|remove_issue), `key`, `target`, `url`, `link_type` | Manage epic links, web links, and issue-to-issue links in one tool. |
| `jira_versions` | `action` (list\|create), `project`, `name`, `release_date`, `description` | List fix versions for a project or create a new version. |

### Confluence

| Tool | Key Parameters | Description |
|------|---------------|-------------|
| `confluence_find` | `query` or `cql`, `space_key`, `content_type`, `limit` | Search pages/blog posts by CQL or natural language; also looks up users. |
| `confluence_get` | `page_id` or `title`+`space_key`, `include_children`, `include_comments`, `include_labels` | Get a page's content plus optional metadata. |
| `confluence_write` | `action` (create\|update\|delete), `space_key`, `title`, `body`, `page_id`, `labels` | Create, update, or delete a page. Manage labels in the same call. |
| `confluence_comment` | `page_id`, `body` | Add a comment to a Confluence page. |

## Tool Access Control

### Enable Specific Tools

Use `ENABLED_TOOLS` environment variable or `--enabled-tools` flag:

```bash
# Environment variable
ENABLED_TOOLS="confluence_find,jira_get,jira_find"

# Command line
uvx mcp-atlassian --enabled-tools "confluence_find,jira_get,jira_find"
```

### Read-Only Mode

Disable all write operations:

```bash
READ_ONLY_MODE=true
```

When enabled, only read operations are available regardless of `ENABLED_TOOLS` setting.
